Online Styleguide

The Online Styleguide is a uniformed compendium for project managers, web publisher, designers or developers of how to maintain, update or further develop a website. It gives an insight of formatting, use of space and components accross the online presence. It is devided into three essential parts.

1. Typography

Within typography we define standards such as text sizes, colors, paddings and dynamic behaviours. This can be included inside a plain text-paragraph or more advanced structures such as tables or forms. Furthermore we add custom styles that support the layout throughout all templates like leading texts, grid support, helpers or list types. We provide a standardized overview of all html functions and essential helpers.

2. Components

Components are part of a website that are used on multiple occurences. Some examples include page navigations, boxes or teasers. Variations of the same element will be provided through descriptive indications. Unlike typography, this section will be updated permanently as components are added or dropped. Besides the live preview we also add code examples to easily encourage copy-and-paste.

3. Templates

This represents an overview of all requested html pages. Those are marked and referenced to through a unique list-numbering. The templates are fully functional in terms of html, css and javascript but do not include any backend logic. Depending on the projects nature, templates might be outdated over time as typography and components advance. They only serve as references to show how they were initially designed.

Structure & naming

Structure

This is the general structure inside the static folder. The asterisk (*) marks folder named css (scss), img or js. For example base.css or base.js. The substructure is applied to all items named above.

static/
??? */
?   ??? includes/
?   ??? libs/
?   ??? plugins/
?   ??? base.ext
??? fonts/
??? swf/

Compiling and minification is automated. Please consult the readme for further instructions.

Naming

All files within static/ or templates/ should be named after the following guidelines.

namespace-module-addition-addition.ext

The namespace is the origin of the file, is unique and always required. In addition, namespaces can be folders to group multiple files and create a better overview. The files inside the folder are excluding the namespace.

Modules are used as soon as the namespace isn't enough to describe the meaning of the file. They are not unique, optional and can be nested in folders.

Additions are additional descriptions for the file like direction, color or position. There can be an infinity amout of additions, they are all divided by a dash.

Exception: We use the "."-annotation for javascript files.

Some examples:

templates/base.html "base" is the namespace
templates/base-sidebar-left.html "base" is the namespace, "sidebar" the module and "left" an addition
img/icon/arrow-left.png "icon" is the namespace and "left" an addition
img/icon/arrow-left-small.png "icon" is the namespace and ""
js/jquery.cycle.min.js "jquery" is the namespace, "cycle" the module and "min" the addition
js/cl.lightbox.js "cl" is the namespace and "lightbox" the module

Plugin guidelines

When developing plugins, the following terms must be fullfilled:

  • W3C HTML validation
  • No JavaScript and Pageload1 optimization
  • Progressive enhancement2
  • Retina support for important brand information like logo

The following requirements should be fullfilled:

  • Full retina support
  • WCAG 2.0 validation

1) When loading HTML and JavaScript, jumping of content elements or other disturbances should be avoided.
2) Wherever possible use HTML5 or CSS3 technologies and provide proper fallbacks.
